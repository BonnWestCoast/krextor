Convenience functions and templates for extracting RDF from XML languages, independent both from the XML input language and from the RDF output notation

This stylesheet provides convenience functions and templates for an RDF extraction from XML languages. It is independent of any RDF output notation.

element()* krextor:literal-properties - source
Defines a simple mapping of XML (child) elements or attributes to ontology properties.

Defines a simple mapping of XML (child) elements or attributes to ontology properties. This mapping is applied to all XML elements or attributes that are processed in krextor:add-literal-property mode.

Elements are given literally inside the XML value of this variable – mind the namespaces! Attributes are given as elements that are named after the attribute and carry an additional krextor:attribute="yes" attribute. For each property, the following details may be provided as attributes:

  • @property – a whitespace-separated list of URIs of properties; see the property parameter of krextor:add-literal-property for details
  • @object-is-list – true if the object is to be interpreted as a whitespace-separated list of multiple objects; see the object-is-list parameter of krextor:add-literal-property for details
  • @normalize-space – true if whitespace around the object is to be normalized; see the property parameter of krextor:add-literal-property for details
  • @datatype – the URI of the datatype of the object; see the property parameter of krextor:add-literal-property for details

The object value is taken from the text or XML content of the element or attribute, i.e. in the same way as when calling the krextor:add-literal-property template without an object parameter.

element()* krextor:resources - source
Defines a simple mapping of XML elements to ontology classes

Defines a simple mapping of XML elements to ontology classes. This mapping is applied to all XML elements that are processed in krextor:create-resource mode.

XML elements are given literally inside the XML value of this variable – mind the namespaces! For each element, the following details may be provided as attributes:

  • @type – a whitespace-separated list of URIs of types of this resource; see the type parameter of krextor:create-resource for details
  • @related-via-properties – a whitespace-separated list of URIs of properties by which the parent resource is related to this resource; see the related-via-properties parameter of krextor:create-resource for details
  • @related-via-inverse-properties – a whitespace-separated list of URIs of properties by which this resource is related to the parent resource; see the related-via-inverse-properties parameter of krextor:create-resource for details

xi:include (mode: krextor:main) - source
Follows an XInclude (generally supported by Krextor)

Krextor supports the following generic inclusion mechanism for XML documents: A root element R of a transcluded documents will be treated like a direct child of the parent element P of the xi:include element. If there is a relevant relationship between P and R, an according triple is generated, with the transcluded document's URI (not the URI of R!) being the object. The transcluded document is loaded and its root node examined in order to find this out. Any relationships between elements of the transcluding document and the transcluded document that are not direct relationships between P and R are not considered during RDF extraction.

Note: We're using XInclude because the semantics of <element xlink:type="simple" xlink:show="embed" xlink:href="some-XML-resource"/> is not yet clearly defined in the XLink specification. Should the root element of the document pointed to replace the pointing element, or should it be transcluded into the pointing element as a child?

krextor:create-resource (param: xs:string? subject, xs:string* related-via-properties, xs:string* related-via-inverse-properties, xs:string* type, node()* properties, node()* process-next, xs:string subject-uri, xs:boolean blank-node, xs:string? this-blank-node-id, xs:boolean collection, xs:boolean nested-call) - source
Creates an RDF resource from the current element

Creates an RDF resource of some type from the current element, and probably creates related triples having this resource as a subject or object. Then, matching extraction templates are applied to the child elements. A call to create-resource defines a scope in which the created resource is the default subject of any other triple created using these templates, unless another resource is created from some child element. The type (i.e. a URI of some class in some ontology) has to be specified.

This resource is assumed the default subject of all triples extracted from descendant elements and attributes using the krextor:add-literal-property and krextor:add-uri-property templates. Optionally, a sequence of properties can be passed by which this resource is related to the “parent” resource, i.e. the resource created by the invoking template (in most cases the resource created from the parent XML element), or vice versa. Additional properties of this resource can be passed, if it is not possible to have them generated by templates matching some attributes or children of this element.

a sequence of properties by which the parent resource is related to this resource a sequence of properties by which this resource is related to its parent resource a sequence of URIs of types of the resource additional properties of this resource, encoded as <krextor:property uri="property-uri+" object="object-uri"/> or <krextor:property uri="property-uri+" value="object-literal"/> or <krextor:property uri="property-uri+">object-literal</krextor:property>. On literal-valued objects, the attributes @language and @datatype are also allowed. Support for blank node objects is not yet implemented. The node set to which further extraction templates are applied (via xsl:apply-templates). The default value is *|@*, i.e. we also process attributes, as they may represent properties of the current resource. true() if this resource is a blank node true() if this resource is an RDF collection
